# QuickTxt.exe 功能设计

## 1. 工具定位

QuickTxt.exe 是一个 Windows 后台常驻小工具，用于通过全局快捷键快速创建并打开一个新的 txt 文件。txt 文件使用当前时间命名，保存目录可配置，并支持查看最近创建或打开的 txt 文件。

## 2. 核心功能结构

```text
QuickTxt.exe
├─ 后台托盘常驻
├─ 全局快捷键，例如 Ctrl + Alt + N
├─ 新建 txt：
│  └─ 2026-06-09_14-32-08.txt
├─ 保存目录可配置
├─ 自动用默认编辑器打开
├─ 托盘菜单：
│  ├─ 新建 txt
│  ├─ 最近 txt
│  ├─ 打开保存目录
│  ├─ 设置
│  └─ 退出
└─ 配置文件：
   └─ %AppData%\QuickTxt\config.json
```

## 3. 推荐技术方案

- 开发语言：C#
- 应用框架：.NET WinForms
- 目标平台：Windows
- 发布形式：单文件 exe 或安装包

选择 WinForms 的原因：

- 适合开发轻量 Windows 桌面工具。
- 内置托盘图标能力：`NotifyIcon`。
- 可通过 Windows API 注册全局快捷键：`RegisterHotKey`。
- 文件创建、配置保存、启动默认编辑器都比较直接。

## 4. 用户流程

### 4.1 后台启动

用户启动 QuickTxt.exe 后，程序不显示主窗口，而是在系统托盘常驻。

启动时程序需要：

- 读取配置文件。
- 确认保存目录存在，不存在则创建或提示用户重新选择。
- 注册全局快捷键。
- 初始化托盘菜单。

### 4.2 快捷键新建 txt

用户按下全局快捷键，例如 `Ctrl + Alt + N`。

程序执行：

1. 获取当前时间。
2. 按命名格式生成文件名，例如 `2026-06-09_14-32-08.txt`。
3. 在配置的保存目录下创建空 txt 文件。
4. 使用系统默认 txt 编辑器打开该文件。
5. 将该文件记录到最近文件列表。

### 4.3 托盘菜单

右键托盘图标显示菜单：

- 新建 txt：立即创建并打开一个新的 txt 文件。
- 最近 txt：显示最近创建或打开过的 txt 文件，点击后打开。
- 打开保存目录：用资源管理器打开配置的保存目录。
- 设置：打开设置窗口。
- 退出：注销快捷键并退出程序。

## 5. 配置设计

配置文件路径：

```text
%AppData%\QuickTxt\config.json
```

建议配置内容：

```json
{
  "saveDirectory": "%USERPROFILE%\\Documents\\QuickTxt",
  "hotkey": {
    "modifiers": ["Ctrl", "Alt"],
    "key": "N"
  },
  "fileNamePattern": "yyyy-MM-dd_HH-mm-ss",
  "openAfterCreate": true,
  "maxRecentFiles": 10
}
```

字段说明：

- `saveDirectory`：txt 文件保存目录。
- `hotkey`：全局快捷键配置。
- `fileNamePattern`：文件名时间格式。
- `openAfterCreate`：创建后是否自动打开。
- `maxRecentFiles`：最近文件显示数量。

## 6. 最近文件设计

最近文件可以采用两种方式实现。

### 方案 A：扫描保存目录

每次打开托盘菜单时，扫描保存目录下最近修改的 `.txt` 文件。

优点：

- 不需要额外维护历史记录。
- 用户手动创建或编辑的 txt 也能显示。

缺点：

- 只能显示当前保存目录下的文件。
- 如果目录内 txt 很多，需要限制扫描数量。

### 方案 B：维护历史记录

创建或打开 txt 时，将文件路径写入最近文件记录。

建议路径：

```text
%AppData%\QuickTxt\recent.json
```

优点：

- 可以准确记录由本工具创建或打开的文件。
- 不依赖目录扫描。

缺点：

- 用户删除文件后，需要清理无效记录。

推荐第一版使用方案 A，简单稳定；后续再增加 `recent.json`。

## 7. 文件命名规则

默认文件名：

```text
yyyy-MM-dd_HH-mm-ss.txt
```

示例：

```text
2026-06-09_14-32-08.txt
```

冲突处理：

- 如果同一秒内重复创建，追加序号。

示例：

```text
2026-06-09_14-32-08.txt
2026-06-09_14-32-08_2.txt
2026-06-09_14-32-08_3.txt
```

## 8. 设置窗口

第一版设置窗口至少包含：

- 保存目录选择。
- 快捷键显示或修改。
- 文件命名格式。
- 最近文件数量。
- 创建后是否自动打开。
- 保存按钮。
- 取消按钮。

后续可增加：

- 开机自启动。
- 托盘单击行为配置。
- 默认编辑器配置。
- 深色模式。

## 9. 异常处理

程序需要处理以下情况：

- 保存目录不存在。
- 保存目录无写入权限。
- 快捷键被其他程序占用。
- 文件创建失败。
- 默认编辑器打开失败。
- 配置文件损坏。
- 最近文件已被删除。

建议策略：

- 非致命错误通过托盘气泡提示或消息框提示。
- 配置损坏时备份旧配置，并生成默认配置。
- 快捷键注册失败时提示用户进入设置修改。

## 10. 开发任务拆分

建议按以下顺序开发：

1. 创建 WinForms 项目。
2. 实现托盘常驻和退出菜单。
3. 实现新建 txt 文件逻辑。
4. 实现自动打开 txt 文件。
5. 实现全局快捷键。
6. 实现配置文件读写。
7. 实现设置窗口。
8. 实现最近文件菜单。
9. 补充异常提示。
10. 发布单文件 exe。

## 11. 第一版最小可用版本

第一版只需要包含：

- 托盘常驻。
- 固定快捷键 `Ctrl + Alt + N`。
- 固定或默认保存目录。
- 按当前时间创建 txt。
- 自动打开 txt。
- 托盘菜单支持新建 txt、打开保存目录、退出。

完成第一版后，再增加配置窗口和最近文件列表。
